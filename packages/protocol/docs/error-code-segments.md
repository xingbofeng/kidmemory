# 错误码分段规则

## 概述

KidMemory 使用数字错误码体系，所有错误码按功能模块分段，便于识别和维护。

## 分段规则

| 范围 | 模块 | 说明 |
|------|------|------|
| 0 | 成功 | API 调用成功 |
| 10000-10999 | 通用错误 | 不属于特定模块的通用错误 |
| 11000-11999 | 鉴权 | 认证、授权相关错误 |
| 12000-12999 | 参数校验 | 请求参数验证失败 |
| 13000-13999 | 素材/书稿 | 素材管理、书稿生成相关错误 |
| 14000-14999 | 分享 | 分享功能相关错误 |
| 15000-15999 | 上传/存储 | 文件上传、存储相关错误 |
| 16000-16999 | 限流/安全 | 限流、安全防护相关错误 |
| 50000+ | 服务端内部错误 | 数据库、外部服务等内部错误 |

## 添加新错误码

1. **确定所属模块**：根据错误类型选择对应的分段范围
2. **检查重复**：确保新错误码在该范围内未被使用
3. **更新代码**：在 `src/common/api-code.ts` 中添加新的枚举值
4. **添加文案**：在 `errors/messages.zh-CN.json` 和 `errors/messages.en-US.json` 中添加对应的错误文案
5. **更新 YAML**：在 `errors/error-codes.yaml` 中记录错误码定义
6. **运行检查**：执行 `npm run check` 确保无重复且文案完整

## 示例

```typescript
// 添加新的上传相关错误码
export enum ApiCode {
  // ... 其他错误码
  
  // 上传/存储 (15000-15999)
  UPLOAD_FILE_TOO_LARGE = 15007,  // 新增
}
```

```json
// errors/messages.zh-CN.json
{
  "15007": "上传文件过大"
}

// errors/messages.en-US.json
{
  "15007": "File too large"
}
```

## 注意事项

- 错误码一旦分配，不应修改或删除，以保持向后兼容
- 每个错误码必须同时提供中英文文案
- 错误码应具有明确的语义，避免歧义
- 优先使用现有错误码，避免创建重复的错误码
